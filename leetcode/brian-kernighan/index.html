位操作 (Bit Manipulation) 往往能通过令人惊叹的技巧带来性能的极致提升。其中，计算一个整数二进制表示中 1 的个数（即 Population Count 或 Hamming Weight）是一个非常经典的问题。

最朴素的解法是循环检查每一位（Loop & Shift），其时间复杂度固定为 $O(n)$。然而，Kernighan’s Algorithm 提供了一种更为优雅且高效的思路，将时间复杂度优化至 $O(k)$（$k$ 为 set bit 的个数）：跳过所有的 0，只处理 1。

核心洞察：n & (n-1)#

Kernighan’s Algorithm 的核心在于一个简单的位运算性质：n & (n-1) 可以消除 n 的二进制表示中最右侧的一个 1 (Least Significant Bit, set bit)。

原理深度剖析#

为了更透彻地理解，我们将一个整数 $n$ 的二进制表示拆分为三个部分来看：

  1. 左侧部分：最右侧 1 左边的所有位（保持不变）。
  2. 关键位：即 $n$ 最右侧的那个 1 (The Rightmost Set Bit)。
  3. 右侧部分：关键位右边的所有 0。

数学表示如下： $$ n = (\text{左侧部分}) + \mathbf{1} + (\underbrace{00\dots0}_{\text{全0}}) $$

第一步：减 1 的连锁反应

当我们计算 $n - 1$ 时，二进制借位（Borrow）机制会从最右侧开始触发，一直向左借位直到遇到关键位 1 为止：

  • 关键位 1：被借位后变成了 0。
  • 右侧部分：原本全是 0，因为借位而全部翻转变成了 1。
  • 左侧部分：未受借位影响，完好无损地保持不变。

第二步：按位与的消除魔术

现在我们将 $n$ 和 $n-1$ 进行按位与（AND）操作，奇迹发生了：

$$ \begin{array}{r c c c} & \text{左侧} & \text{关键位} & \text{右侧} \\ n & = \dots \mathbf{X} & 1 & 00\dots0 \\ n - 1 & = \dots \mathbf{X} & 0 & 11\dots1 \\ \hline n \ \& \ (n-1) & = \dots \mathbf{X} & \mathbf{0} & \mathbf{00\dots0} \end{array} $$

结果分析：

  • 左侧：X & X = X（保持原样）。
  • 关键位：1 & 0 = 0（被消除）。
  • 右侧：0 & 1 = 0（依旧是 0）。

最终结论：整个操作唯一改变的，就是精准地“抹去”了最右侧的那个 1，而保持其他所有位不动。这就是为什么循环次数只取决于 1 的个数的原因。

复杂度分析#

通常的移位算法需要遍历整数的所有位，例如对于 32 位整数，无论数值大小都需要循环 32 次。

而 Kernighan’s Algorithm 的循环次数 仅等于 1 的个数（Set Bits Count）。

  • 时间复杂度：$O(k)$，其中 $k$ 为 set bits 的数量。在最坏情况下（全为 1）与普通算法持平，但在大多数场景下（稀疏的 1）效率显著更高。
  • 空间复杂度：$O(1)$。

func hammingWeight(n int) int {
    count := 0
    // 只要 n 不为 0，说明还有至少一个 1
    for n > 0 {
        // 消除最右侧的一个 1
        n = n & (n - 1)
        count++
    }
    return count
}

典型应用场景#

除了基础的位计数，n & (n-1) 这一核心性质在实际工程中还有多种巧妙应用：

  1. 判断 2 的幂 (Power of Two) 2 的幂在二进制中有且仅有一个 1。利用此特性，使用 n > 0 && (n & (n - 1)) == 0 即可在 $O(1)$ 时间内完成判断，比循环除以 2 快得多。

  2. 计算汉明距离 (Hamming Distance) 汉明距离即两个数字二进制位不同的个数。计算 x ^ y（异或）后，实际上就是计算结果中 1 的个数，此时 Kernighan’s Algorithm 便是最佳搭档。

  3. 稀疏位图 (Sparse Bitset) 迭代 在处理稀疏集合（BitMap）时，使用此算法可以“跳跃式”地遍历所有元素，直接定位到设值的位，避免了低效的逐位扫描。

延伸思考#

虽然 Kernighan’s Algorithm 在软件层面已经非常高效，但在现代高性能计算中，许多 CPU 指令集（如 x86 的 POPCNT 指令）已经在硬件层面直接支持了 Population Count 操作，能以单指令周期完成计算。但在不支持特定指令集的硬件或跨平台软件实现中，Kernighan’s Algorithm 依然是不可或缺的利器。
